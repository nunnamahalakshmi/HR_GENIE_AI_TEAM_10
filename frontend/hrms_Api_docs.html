Overview

Getting started

The HRMS REST API allows developers to integrate employee management, payroll, and attendance data into external tools and systems. All API requests must be made over HTTPS. Responses are returned in JSON format.

Base URL: https://api.hrmsportal.com/v1

Security

Authentication

All API requests require a Bearer token in the Authorization header. Obtain a token by calling the /auth/login endpoint.

HTTP Header
Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json
Endpoints

API Reference

POST /auth/login

Authenticate a user and receive an access token for subsequent API calls.

Request Body

ParameterTypeRequiredDescription
emailstringRequiredEmployee's registered email address
passwordstringRequiredAccount password (min 8 characters)
Request
POST /v1/auth/login
{
  "email": "nikesh@company.com",
  "password": "MyPassword123"
}
Response 200
{
  "status": "success",
  "token": "eyJhbGciOiJIUzI1NiIsInR5...",
  "role": "HR",
  "name": "Nikesh Kumar",
  "expires_in": 3600
}
GET /employees

Retrieve a list of all employees. Requires HR Admin role.

Query Parameters

ParameterTypeRequiredDescription
departmentstringOptionalFilter by department name
rolestringOptionalFilter by role: HR or Employee
pageintegerOptionalPage number for pagination (default: 1)
limitintegerOptionalResults per page (default: 20, max: 100)
Response 200
{
  "total": 1248,
  "page": 1,
  "employees": [
    {
      "id": "EMP001",
      "name": "Nikesh Kumar",
      "email": "nikesh@company.com",
      "department": "Engineering",
      "role": "Employee",
      "status": "Active"
    }
  ]
}
GET /employees/:id

Retrieve full profile details of a single employee by their employee ID.

Example Request
GET /v1/employees/EMP001
Response 200

  "department": "Engineering",
  "designation": "Senior Developer",
  "joining_date": "2023-06-01",
  "status": "Active",
  "salary": 85000
}
GET /attendance

Retrieve attendance records filtered by employee and date range.

ParameterTypeRequiredDescription
employee_idstringRequiredThe employee's ID
fromdateRequiredStart date (YYYY-MM-DD)
todateRequiredEnd date (YYYY-MM-DD)
Response 200
{
  "employee_id": "EMP001",
  "records": [
    {
      "date": "2026-03-22",
      "check_in": "09:02 AM",
      "check_out": "06:15 PM",
      "hours": 9.22,
      "status": "Present"
    }
  ]
}
POST /leave/apply

Submit a leave application for an employee.

ParameterTypeRequiredDescription
employee_idstringRequiredThe employee's ID
leave_typestringRequiredcasual / medical / earned
from_datedateRequiredLeave start date (YYYY-MM-DD)
to_datedateRequiredLeave end date (YYYY-MM-DD)
reasonstringOptionalReason for leave
Request
POST /v1/leave/apply
{
  "employee_id": "EMP001",
  "leave_type": "casual",
  "from_date": "2026-04-10",
  "to_date": "2026-04-11",
  "reason": "Personal work"
}
Response 200
{
  "status": "success",
  "leave_id": "LV20260410001",
  "message": "Leave application submitted for approval"
}

ParameterTypeRequiredDescription

  "deductions": {
    "pf": 5100,
